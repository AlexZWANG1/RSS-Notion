## 阶段 1：拉取与筛选 RSS

### 1.1 获取兴趣关键词

1. 使用 `notion-fetch` 读取 AI助手配置页（page ID: `{{CONFIG_PAGE_ID}}`），提取：
   - 长期关注课题列表
   - 关键词表
   - 是否有"指定课题"（如果有，记录下来供阶段 2 使用）
2. 使用 `notion-query-database-view` 查询研究文档库（data_source: `{{RESEARCH_DATA_SOURCE}}`），提取已有课题的标题作为补充关键词

### 1.2 拉取 RSS 文章

使用 folo-mcp 的 `entry_list` 工具拉取文章：
- 参数：`read: {{READ_MODE}}`，`limit: {{MAX_RSS_ARTICLES}}`，`withContent: false`（只要标题和摘要，不要全文）
- 如果返回 0 篇文章，跳到阶段 2

### 1.3 AI 评分筛选

对每篇文章，基于标题和摘要（description），结合 1.1 获取的关键词和课题列表，评分 1-10：
- 10: 直接命中长期关注课题的核心内容
- 7-9: 与关注领域高度相关
- 4-6: 有一定相关性但不是核心关注
- 1-3: 不相关

输出评分结果列表（内部使用，不写入 Notion）：
```
标题 | 评分 | 话题分类 | 媒体来源 | URL
```

**媒体来源提取**：从 RSS feed 名称或文章来源中提取媒体/博主名称（如 "36氪"、"Simon Willison's Weblog"、"The Verge" 等）。如果 entry_list 返回的数据中包含 feed 信息，使用 feed 标题作为媒体来源。

### 1.4 获取入选文章的完整内容

筛选评分 >= {{RELEVANCE_THRESHOLD}} 的文章，最多 {{MAX_SELECTED}} 篇。

对每篇入选文章，获取完整内容：

**普通文章**：使用 `entry_list` 工具，传入该文章的 `entryId`，设置 `withContent: true`，获取文章全文。

**YouTube 视频**：如果文章 URL 包含 `youtube.com` 或 `youtu.be`：
1. 从 URL 中提取视频 ID（如 `v=xxx` 或 `youtu.be/xxx`）
2. 使用 Bash 工具执行 `curl -s "http://localhost:3001/api/transcript?v={videoId}"` 获取字幕
3. 如果字幕获取成功，基于字幕生成一篇 500-1000 字的中文视频摘要，结构：
   - **核心观点**（3-5 个要点）
   - **详细内容**（关键论述和数据）
   - **值得关注的引用/金句**
4. 如果字幕获取失败（服务未运行或视频无字幕），退回使用文章描述

### 1.5 写入收件箱

**写入前去重**：对每篇入选文章，先使用 `notion-search` 在收件箱中搜索是否已有同标题的条目。如果已存在，跳过该文章。

对每篇入选文章，使用 `notion-create-pages` 写入收件箱（data_source: `{{INBOX_DATA_SOURCE}}`）：
- 名称: YouTube 视频用 `[视频摘要] <标题>`，普通文章用 `[AI精选] <文章标题>`
- 来源: YouTube 视频用 `视频摘要`，普通文章用 `RSS精选`
- 状态: `未读`
- 话题: 根据评分时的话题分类填写（如 `Agent Infra`、`云原生` 等）
- 原文链接: 文章/视频 URL
- 收录时间: {{TODAY}}
- 重要性: 评分 9-10 为 `高`，7-8 为 `中`
- 媒体来源: RSS feed 名称/博主名称（如 `No Priors`、`a16z`、`The Verge` 等）
- 页面内容:
  - YouTube 视频：AI 生成的视频摘要（基于字幕）
  - 普通文章：文章全文（从 `withContent: true` 获取）。如果全文过长（超过 3000 字），截取前 3000 字并注明"[全文已截断，点击原文链接查看完整内容]"

如果单篇文章写入失败，记录错误并继续处理下一篇，不要中断整个流程。

### 1.6 标记已读

使用 `mark_read` 工具标记所有已处理的文章为已读（不仅是入选的，而是所有拉取到的）。

## 阶段 2：每日课题研究

### 2.1 确定课题

1. 如果阶段 1.1 中发现了"指定课题"→ 使用该课题
2. 如果没有指定课题，且阶段 1 筛选出了文章 → 从入选文章中提取出现频率最高的主题作为课题
3. 如果没有指定课题，且阶段 1 筛选出 0 篇文章 → 跳过本阶段，记录"今日无课题"

### 2.2 生成研究报告

（200-500 字）

（500-1000 字）

（300-500 字）

- 标注哪些是今日 RSS 中的内容

## 重要规则

1. **错误处理**：单篇文章处理失败不要中断整个流程，记录错误继续执行
2. **去重**：写入任何内容前，先搜索是否已存在同标题条目
3. **AI 标注**：所有 AI 生成的内容必须有明确标注（标题前缀 + 来源属性 + 正文标注）
4. **Token 效率**：评分阶段用 `withContent: false` 只用标题和摘要；写入阶段才对入选文章拉全文（`withContent: true`）或提取 YouTube 字幕
5. **顺序执行**：必须按阶段 1→2→3 顺序执行，不要跳过或并行
